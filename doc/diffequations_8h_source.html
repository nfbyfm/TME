258 /*************************************************************************
259 Cash-Karp adaptive ODE solver.
260 
261 This subroutine solves ODE Y'=f(Y,x) with initial conditions Y(xs)=Ys
262 (here Y may be single variable or vector of N variables).
263 
264 INPUT PARAMETERS:
265  Y - initial conditions, array[0..N-1].
266  contains values of Y[] at X[0]
267  N - system size
268  X - points at which Y should be tabulated, array[0..M-1]
269  integrations starts at X[0], ends at X[M-1], intermediate
270  values at X[i] are returned too.
271  SHOULD BE ORDERED BY ASCENDING OR BY DESCENDING!
272  M - number of intermediate points + first point + last point:
273  * M>2 means that you need both Y(X[M-1]) and M-2 values at
274  intermediate points
275  * M=2 means that you want just to integrate from X[0] to
276  X[1] and don't interested in intermediate values.
277  * M=1 means that you don't want to integrate :)
278  it is degenerate case, but it will be handled correctly.
279  * M<1 means error
280  Eps - tolerance (absolute/relative error on each step will be
281  less than Eps). When passing:
282  * Eps>0, it means desired ABSOLUTE error
283  * Eps<0, it means desired RELATIVE error. Relative errors
284  are calculated with respect to maximum values of Y seen
285  so far. Be careful to use this criterion when starting
286  from Y[] that are close to zero.
287  H - initial step lenth, it will be adjusted automatically
288  after the first step. If H=0, step will be selected
289  automatically (usualy it will be equal to 0.001 of
290  min(x[i]-x[j])).
291 
292 OUTPUT PARAMETERS
293  State - structure which stores algorithm state between subsequent
294  calls of OdeSolverIteration. Used for reverse communication.
295  This structure should be passed to the OdeSolverIteration
296  subroutine.
297 
298 SEE ALSO
299  AutoGKSmoothW, AutoGKSingular, AutoGKIteration, AutoGKResults.
